Overzicht

1. Projectstructuur

Dit is hoe de bestanden georganiseerd zijn. Elke map en elk bestand heeft een eigen taak:

pwa/ ├── server.js ← Node.js server, draait op je pc ├── package.json ← lijstje van npm-pakketten └── public/ ← alles hierin is bereikbaar via de browser ├── index.html ← de homepagina ├── invoer.html ← formulierpagina ├── overzicht.html ← lijstpagina ├── manifest.json ← maakt de app installeerbaar ├── sw.js ← service worker voor offline ├── css/ │ └── style.css ← alle opmaak ├── js/ │ ├── opslag.js ← localStorage lezen/schrijven │ ├── taal.js ← NL/EN wisselen │ ├── home.js ← dashboard logica │ ├── invoer.js ← formulier verwerken │ └── overzicht.js ← filteren en resetten ├── locales/ │ ├── nl.json ← Nederlandse teksten │ └── en.json ← Engelse teksten └── icons/ ├── icon-192.png └── icon-512.png
Tip: De map public/ is speciaal. Alles wat daarin staat, kan de browser openen. Bestanden buiten public/ (zoals server.js) zijn niet direct bereikbaar vanuit de browser — dat is veiliger.
Node.js + Express

2. server.js – de Express server

De server is de motor van de app. Hij draait op je pc en stuurt bestanden naar de browser als iemand een pagina opvraagt.

const express = require('express');
const path    = require('path');

const app = express();

app.use(express.static(path.join(__dirname, 'public')));

app.listen(3000, function() {
  console.log('Server draait op http://localhost:3000');
});

Wat doet elke regel?

CodeWat het doet
require('express')Laadt het Express pakket in. Zoals een import in Python.
require('path')Hulpmiddel voor bestandspaden, werkt op Windows én Mac.
express()Maakt een nieuwe server aan en slaat die op in app.
express.static(...)Zegt: "alles in de map public mag de browser downloaden".
app.listen(3000)Start de server op poort 3000. Poort = een "deur" op je pc.

Wat kun je nog meer met Express?

Express kan veel meer dan alleen bestanden sturen. Bijvoorbeeld:

// Een URL aanmaken die data teruggeeft
app.get('/api/hallo', function(req, res) {
  res.json({ bericht: 'Hallo wereld!' });
});

// Data ontvangen die iemand opstuurt (bijv. een formulier)
app.post('/api/opslaan', function(req, res) {
  console.log(req.body); // de verstuurde data
});
Onthoud: req = wat de browser vraagt, res = wat de server terugstuurt.

Semantische HTML-elementen

Je gebruikt "semantische" elementen. Dat betekent: de naam van het element vertelt wat erin zit. Dit is beter dan overal <div> gebruiken.

data-i18n — wat is dat?

Je ziet op veel elementen data-i18n="...". Dit is een zelfgemaakt attribuut dat taal.js gebruikt om te weten welke tekst vervangen moet worden bij een taalwisseling.

<!-- "nav.home" verwijst naar de sleutel in nl.json of en.json -->
<a href="/index.html" data-i18n="nav.home">🏠 Home</a>
Tip: Je kunt zelf ook data- attributen verzinnen voor elk doel. Bijv. data-id="5" om een ID op te slaan bij een knop.
CSS

4. style.css – de opmaak

De CSS maakt de app er mooi uit zien. Het is geschreven "mobile-first": je schrijft eerst de stijl voor kleine schermen, en dan pas voor grote schermen met @media.

De basis: reset en body

/* Verwijder standaard marges van de browser */
* {
  box-sizing: border-box;  /* padding telt mee in de breedte */
  margin: 0;
  padding: 0;
}

body {
  font-family: Arial, sans-serif;
  background: #f5f5f5;
  color: #222;
}

Grid – kaarten naast elkaar

/* Standaard: 2 kolommen op mobiel */
.kaarten {
  display: grid;
  grid-template-columns: 1fr 1fr;  /* 2 kolommen van gelijke breedte */
  gap: 12px;
}

/* Op schermen breder dan 600px: 4 kolommen */
@media (min-width: 600px) {
  .kaarten {
    grid-template-columns: repeat(4, 1fr);
  }
}
Wat is 1fr? "fr" staat voor "fractie". 1fr 1fr betekent: verdeel de ruimte in 2 gelijke stukken. repeat(4, 1fr) doet hetzelfde maar dan 4 keer.

Flexbox – dingen naast elkaar zetten

header {
  display: flex;               /* zet kinderen naast elkaar */
  justify-content: space-between; /* één links, één rechts */
  align-items: center;         /* verticaal gecentreerd */
}

Flexbox vs Grid: gebruik Flexbox voor één rij of kolom, gebruik Grid voor een heel raster.

@media query – responsive design

/* Dit geldt ALTIJD (mobiel) */
nav a {
  font-size: 0.9rem;
}

/* Dit geldt alleen op brede schermen */
@media (min-width: 600px) {
  nav a {
    font-size: 1rem;
  }
}
JavaScript

5. opslag.js – localStorage

LocalStorage is een opslagplek in de browser. Het werkt zoals een klein kladblok dat de browser bijhoudt, ook als je de pagina sluit. Er is geen server of database voor nodig.

Hoe werkt localStorage?

LocalStorage slaat alles op als tekst. Maar wij willen lijstjes (arrays) en objecten opslaan. Daarvoor gebruiken we JSON.stringify (object → tekst) en JSON.parse (tekst → object).

// Stel je voor: dit is een item
var item = { datum: '2025-06-10', categorie: 'voeding', waarde: 500 };

// Opslaan: object → tekst
localStorage.setItem('mijnSleutel', JSON.stringify(item));

// Ophalen: tekst → object
var opgehaald = JSON.parse(localStorage.getItem('mijnSleutel'));

// Verwijderen
localStorage.removeItem('mijnSleutel');

getItems() – alle items ophalen

function getItems() {
  var data = localStorage.getItem('gezondheid');
  if (data) {
    return JSON.parse(data);  // tekst omzetten naar array
  }
  return [];  // als er niks is: geef een lege array terug
}

itemToevoegen() – een item opslaan

function itemToevoegen(item) {
  var items = getItems();   // haal de huidige lijst op
  item.id = Date.now();   // geef het item een uniek getal als ID
  items.push(item);       // voeg het toe aan de lijst
  saveItems(items);       // sla de hele lijst weer op
}
Wat is Date.now()? Dit geeft het aantal milliseconden terug sinds 1 januari 1970. Het is altijd een uniek getal, dus perfect als ID.

itemResetten() – waarde op 0 zetten

function itemResetten(id) {
  var items = getItems();
  for (var i = 0; i < items.length; i++) {
    if (items[i].id == id) {      // zoek het juiste item
      items[i].waarde = 0;        // zet waarde op 0
    }
  }
  saveItems(items);              // sla op
}

De for-lus loopt door de hele lijst heen. i is een teller die begint bij 0. items[i] is het item op plek i.

Wat kun je nog meer met localStorage?

// Alle sleutels bekijken
for (var i = 0; i < localStorage.length; i++) {
  console.log(localStorage.key(i));
}

// Controleren of iets bestaat
if (localStorage.getItem('gebruikersnaam')) {
  console.log('Gebruiker is bekend');
}

6. invoer.js – het formulier

Dit bestand regelt alles rondom het invoerformulier: datum instellen, valideren en opslaan.

Datum automatisch invullen

var vandaag = new Date().toISOString().split('T')[0];
document.getElementById('datum').value = vandaag;

Stap voor stap:

new Date() → maakt een datum-object van nu: bijv. Mon Jun 10 2025 14:30:00
.toISOString() → maakt er een tekst van: "2025-06-10T14:30:00.000Z"
.split('T') → knipt op de T: ["2025-06-10", "14:30:00.000Z"]
[0] → pakt het eerste deel: "2025-06-10"

Luisteren naar events

document.getElementById('categorie')
  .addEventListener('change', function() {
    // dit wordt uitgevoerd als de dropdown verandert
    console.log(this.value); // 'this' = het element zelf
  });

addEventListener is de manier om te reageren op wat een gebruiker doet. Veelgebruikte events:

EventWanneer wordt het getriggerd?
clickAls iemand ergens op klikt
submitAls een formulier wordt verstuurd
changeAls een dropdown of input verandert
inputElke keer als iemand typt
keydownAls een toets ingedrukt wordt

Formulier valideren

document.getElementById('gezondheidForm')
  .addEventListener('submit', function(e) {

  e.preventDefault(); // voorkom dat de pagina herladen wordt

  var omschrijving = document.getElementById('omschrijving').value.trim();

  if (!omschrijving) {
    // veld is leeg: toon foutmelding
    document.getElementById('foutmelding').style.display = 'block';
    return; // stop de functie hier
  }

  // alles ok: ga verder
});
Wat doet .trim()? Het verwijdert spaties aan het begin en einde van een tekst. Zo telt " " (alleen spaties) niet als een geldige invoer.

7. overzicht.js – filteren en resetten

Dit bestand toont de opgeslagen items en filtert ze op dag, week of maand.

Filteren op periode

function toonItems(periode) {
  var items = getItems();
  var nu = new Date();
  var gefilterd = [];

  for (var i = 0; i < items.length; i++) {
    var itemDatum = new Date(items[i].datum);

    if (periode === 'week') {
      var verschil = (nu - itemDatum) / (1000 * 60 * 60 * 24);
      // 1000ms × 60s × 60min × 24u = 1 dag in milliseconden
      if (verschil >= 0 && verschil < 7) {
        gefilterd.push(items[i]);
      }
    }
  }
}
Waarom die berekening? JavaScript rekent datums in milliseconden. Door het verschil te delen door het aantal milliseconden in een dag, krijg je het verschil in dagen.

Dynamisch HTML aanmaken

var lijst = document.getElementById('itemsLijst');
lijst.innerHTML = ''; // maak de lijst leeg

for (var i = 0; i < gefilterd.length; i++) {
  var li = document.createElement('li'); // maak een nieuw <li> element
  li.textContent = gefilterd[i].omschrijving;
  lijst.appendChild(li); // voeg toe aan de lijst
}

Dit is een belangrijk concept: je maakt HTML-elementen met JavaScript en plaatst ze in de pagina. Zo hoef je niet voor elk item een regel HTML te schrijven.

Een knop toevoegen aan elk item

var resetKnop = document.createElement('button');
resetKnop.textContent = '↩ Reset';
resetKnop.setAttribute('data-id', item.id); // sla het ID op bij de knop

resetKnop.onclick = function() {
  var id = this.getAttribute('data-id'); // haal het ID op
  itemResetten(id);
  toonItems(huidigePeriode); // ververs de lijst
};

li.appendChild(resetKnop);

8. taal.js – de taalswitch

De taalswitch laadt een JSON-bestand met vertalingen en past alle teksten op de pagina aan.

Hoe zit een JSON-bestand eruit?

// nl.json
{
  "nav": {
    "home": "🏠 Home",
    "invoer": "➕ Invoer"
  },
  "form": {
    "opslaan": "Opslaan"
  }
}

In de HTML staat dan: data-i18n="nav.home". De punt scheidt het "hoofdstuk" van de "sleutel".

fetch() – een bestand ophalen

function taalLaden(taal) {
  fetch('/locales/' + taal + '.json')  // vraag het bestand op
    .then(function(res) {
      return res.json();   // zet de tekst om naar een object
    })
    .then(function(data) {
      vertalingen = data;  // sla de vertalingen op
      taalToepassen();     // pas alle teksten aan
    });
}
Wat is .then()? fetch is asynchroon — het duurt even voor het bestand binnenkomt. .then() zegt: "als het klaar is, doe dan dit". De ene .then geeft de data door aan de volgende.

Alle teksten vervangen

function taalToepassen() {
  // zoek alle elementen met een data-i18n attribuut
  var elementen = document.querySelectorAll('[data-i18n]');

  for (var i = 0; i < elementen.length; i++) {
    var sleutel = elementen[i].getAttribute('data-i18n'); // bijv. "nav.home"
    var tekst   = vertaling(sleutel);                     // zoek de vertaling op
    if (tekst) {
      elementen[i].textContent = tekst;                    // vervang de tekst
    }
  }
}

querySelectorAll('[data-i18n]') werkt als een CSS-selector. Je kunt hier elk CSS-patroon gebruiken, bijv. '.kaart' of '#mijnId'.

PWA

9. sw.js – offline werken

Een service worker is een apart stukje JavaScript dat op de achtergrond draait. Het werkt als een tussenpersoon tussen de browser en het internet.

// Stap 1: registreer de service worker vanuit je pagina
if ('serviceWorker' in navigator) {
  navigator.serviceWorker.register('/sw.js');
}

Installatie – bestanden opslaan in cache

var CACHE_NAAM = 'gezondheid-v1';
var BESTANDEN  = ['/index.html', '/css/style.css', '/js/opslag.js'];

self.addEventListener('install', function(e) {
  e.waitUntil(
    caches.open(CACHE_NAAM).then(function(cache) {
      return cache.addAll(BESTANDEN); // download en sla op
    })
  );
});

Fetch – gebruik cache als er geen internet is

self.addEventListener('fetch', function(e) {
  e.respondWith(
    caches.match(e.request).then(function(cached) {
      return cached || fetch(e.request);
      // gebruik de cache — of als die er niet is: haal het van internet
    })
  );
});
Hoe test je offline? Open Chrome DevTools (F12) → tabblad "Application" → "Service Workers" → zet "Offline" aan. Ververs de pagina. Als alles goed is, werkt de app nog steeds.

10. manifest.json – installeerbaar maken

Het manifest vertelt de browser hoe de app eruit moet zien als iemand hem installeert op zijn telefoon of pc.

{
  "name":             "Mijn Gezondheid",      // volledige naam
  "short_name":       "Gezondheid",          // naam op het homescreen
  "start_url":        "/index.html",         // welke pagina opent bij starten
  "display":          "standalone",          // zonder browser-balk tonen
  "background_color": "#ffffff",             // kleur tijdens laden
  "theme_color":      "#4CAF50",             // kleur van de statusbalk
  "icons": [
    { "src": "/icons/icon-192.png", "sizes": "192x192" },
    { "src": "/icons/icon-512.png", "sizes": "512x512" }
  ]
}
VeldWat het doet
display: standaloneApp opent zonder adresbalk, ziet eruit als een echte app
display: browserOpent gewoon in de browser
display: fullscreenVolledig scherm, zonder statusbalk (voor games)
theme_colorKleur van de balk bovenaan op Android
Controleren: open Chrome DevTools → tabblad "Application" → "Manifest". Hier zie je of alles correct geladen is en kun je de installeerknop testen.
